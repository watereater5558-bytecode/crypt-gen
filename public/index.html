How this is actually secure

  • The connection is wss:// — TLS-encrypted WebSocket — so nothing on the wire is readable in transit.
  • Generation happens with Node's crypto module on a serverless function that holds no state between connections.
  • Every Ed25519 keypair is signature-tested against itself server-side before it's returned — a keypair that can't sign and verify a nonce is never shown to you.
  • Nothing is persisted. Closing this tab is the same as the key never having existed on this server.
  • Treat this as a convenience tool, not a vault: for production secrets, generate on a machine you trust and store them in a proper secrets manager.
